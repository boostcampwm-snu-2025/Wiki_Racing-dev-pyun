# 🏁 WikiRacing (위키레이싱)
Wiki 링크를 타고 목적지까지 도달하라! > 방대한 지식의 바다에서 길을 찾는 재미, 깔끔한 UI와 시각화된 경로로 즐기는 새로운 위키레이싱 게임입니다.

## 📖 Wiki
이 프로젝트의 상세한 기획 및 디자인, 구현 과정은 [Wiki](https://github.com/boostcampwm-snu-2025/Wiki_Racing-dev-pyun/wiki)에서 확인하실 수 있습니다.

## 📝 프로젝트 소개
**위키레이싱(WikiRacing)**은 시작 문서에서 출발하여, 문서 내에 존재하는 내부 링크(Hyperlink)만을 클릭해 목표 문서에 도달하는 것을 목표로 하는 게임입니다. 기존 위키 사이트에서 느낄 수 있는 피로감을 줄이고, n8n 스타일의 직관적인 노드 시각화와 Skeleton UI를 도입하여 깔끔하고 몰입감 있는 사용자 경험을 제공하고자 합니다.

## 🎯 서비스 목표
- 시각적 즐거움: 텍스트 위주의 위키 탐색에서 벗어나, 링크 간의 관계를 트리(Tree) 구조로 시각화하여 탐색의 재미를 더합니다.

- 경쟁의 재미: 리더보드를 통해 다른 사용자들과 경로 및 점수를 경쟁할 수 있습니다.

- 지식의 연결: 문서와 문서가 어떻게 연결되어 있는지 파악하며 의외의 연관성을 발견하는 즐거움을 제공합니다.

## 🛠 사용 기술 스택 (Tech Stack)
Wiki 기획 문서를 바탕으로 선정된 주요 기술 스택입니다.

### Frontend
- React: 사용자 인터페이스 구현 및 컴포넌트 구조화
- TailwindCSS: 직관적이고 깔끔한 스타일링 및 레이아웃 구성
- Skeleton UI: 데이터 로딩 시 사용자 경험(UX) 개선을 위한 UI 패턴 적용
- Visualization: 링크 관계 시각화 (Tree 구조 등)

### Logic & AI
- Word2Vec: 문서 간의 유사도 분석 및 게임 난이도 산정 로직에 활용
- Pathfinding Algorithm: 최적 경로 탐색 및 난이도 밸런싱 (참고용)

🎮 게임 규칙
위키레이싱의 본질적인 재미를 위해 다음과 같은 제약 사항이 존재합니다.

- **외부 검색 금지**: 구글, 네이버 등 외부 검색 엔진 사용 불가
- **URL 조작 금지**: 주소창을 직접 수정하여 이동 불가
- **내부 검색 금지**: 위키 사이트 내 제공되는 검색 기능 사용 불가
- **뒤로가기 제한**: 브라우저의 뒤로가기 기능을 사용할 수 없음 (Stack 구조로 경로 기록)
- **오직 링크만 사용**: 문서 본문에 포함된 하이퍼링크만 클릭 가능

✨ 주요 기능 (Key Features)
1. 직관적인 게임 플레이 (Game Play)
- 트리(Tree) 뷰: 현재 문서에서 갈 수 있는 링크들을 자식 노드로 시각화하여 보여줍니다.

- 노드 선택: 링크를 선택하면 해당 노드가 강조되며 다음 단계의 링크들이 펼쳐집니다.

2. 리더보드 (Leaderboard)
- 랭킹 시스템: 1위부터 10위까지의 플레이어 기록 제공

- 상세 정보: 닉네임, 시작/종료 문서, 난이도, 점수 표시

- 경로 보기: 다른 플레이어가 이동한 경로를 마인드맵 형식으로 시각화하여 복기 가능

3. 난이도 및 편의성
- 난이도 분석: Word2Vec 등을 활용해 적절한 난이도의 게임 제공

- 고립 문서 배제: 링크가 없어 진행이 불가능한 문서를 배제하여 게임 흐름 끊김 방지

- 연습/도전 모드: 자유롭게 연습하거나 랜덤 매칭으로 도전하는 모드 구분

## 🎨 디자인 (Design)
- 사용자의 피로도를 줄이기 위해 TailwindCSS를 활용한 심플하고 모던한 UI를 지향합니다.

- Loading: Skeleton UI를 적용하여 쾌적한 로딩 경험 제공

- Visual: 복잡한 텍스트 대신 n8n 워크플로우와 유사한 노드 기반의 링크 시각화
